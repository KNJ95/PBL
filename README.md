# Be-Ready 評価 POC アプリ

北海学園大学 佐藤教授との共同プロジェクト「Be-Ready人材育成プログラム」における評価軸の実証実験（POC）用 Web アプリ。

**コンセプト**: アンケート（自己評価）+ ログ（YWT）+ 振り返り の3要素で、Be-Ready人材の評価基準を体験・検証する。

## 1. プロダクト概要

**作成日**: 2026年5月26日  
**最終更新**: 2026年7月9日（チュートリアル・ルーブリックポップアップ・日次/マイルストーン区分・②⑧本採点復帰）  
**バージョン**: v1.5（POC）  
**対象ユーザー**: 学生 / メンター

### 1.1 背景

座学教育との差別化を目的に、企業が「即戦力かどうか」を判断できる評価軸（9項目）を定義。その評価軸を実際に運用・検証するための POC ツールとして開発。

### 1.2 評価の3要素

| 要素 | 内容 | 評価への寄与 |
|---|---|---|
| アンケート | 9評価軸 × 記述式4択（`survey_questions.json` 定義）による自己判定 | 定量的スコア・成長可視化 |
| ログ | YWT法（やったこと / わかったこと / 次にやること）による振り返り | 思考プロセスの記録 |
| 振り返り | メンチメーター形式 / アンケート形式の2モード提出 | 成長実感の言語化・メンターへの提出 |

### 1.3 評価軸9項目（0626ルーブリック準拠）

| 番号 | 軸名 | 評価すること | 分類 |
|---|---|---|---|
| ① | 課題設定力 | 仕事について自分の視点からその価値・意味を理解しているか | A：思考・判断系 |
| ② | 情報活用力 | 必要な情報を収集・整理し、意思決定や行動に活かせるか | A：思考・判断系 |
| ③ | 不確実性への耐性 | 不確実性の高い仕事でも粘り強くかつ柔軟に取り組めるか | A：思考・判断系 |
| ④ | 提案・発信力 | やるべき・やりたいと考えることを他者に伝えられるか | B：行動・実行系 |
| ⑤ | 実行・改善力 | 他でもない自分のこととして粘り強く関われるか | B：行動・実行系 |
| ⑥ | オーナーシップ | やりたいと思っているか、意味を自分なりに理解できているか | B：行動・実行系 |
| ⑦ | 協働・調整力 | 他者の意見を取り入れつつ柔軟に対応できるか | C：関係・姿勢系 |
| ⑧ | 自律・内発的動機 | 自分なりの理由・動機を持ってやりたいと語れるか | C：関係・姿勢系 |
| ⑨ | 行動変容力 | フィードバックを行動の変化につなげられるか | C：関係・姿勢系 |

> **v1.5変更点**: v1.4で「参考値」扱いにしていた②⑧を本採点に復帰。0626ルーブリックには②⑧の詳細基準が未記載のため `RUBRIC_DATA` では `levels: null`（ポップアップに「基準未確定」と表示）。採点上は他軸と同等に扱う。

### 1.4 評価レベル（4分類・0626ルーブリック準拠）

| Lv | 名称 | 理解の種類 | 定義 |
|---|---|---|---|
| Lv.1 | 受動性 | 「知る」理解 | 対象を外から（外在的視点）見て、傍観者的・人ごと的にただ知っているだけの状態 |
| Lv.2 | 能動性 | 「分かる」理解（他者視点） | 他者の立場に立てば確かにそう思える、となっている状態 |
| Lv.3 | 自律性 | 「分かる」理解（自己視点） | 他者ではなく、自分の視点から見て確かにそう思える、となっている状態 |
| Lv.4 | 創造性 | 「発見」としての理解 | 教え手から学ぶのではなく、自らの行為・経験から新たにやりたい・やるべきことを見つけ出している |

### 1.5 佐藤教授による概念整理論点（2026/6/26）

次回ディスカッションで整理予定の未解決論点。アプリの評価軸の扱いに影響する可能性あり。

1. **②情報活用力の次元**: 他軸が「仕事の理解」を見るのに対し、②は「知識・情報の理解」を見ており次元が異なる
2. **⑧内発的動機の省略可能性**: ⑧は行動レベルでなく「理解レベルそのもの」であり、他項目も結局は内発的動機を測っているため省略の余地あり
3. **創造性の独立性**: 受動・能動・自律は「与えられた仕事の自分事化」だが、創造性は「自ら仕事を生み出す」→ 4つを一軸に並べる妥当性に留保
4. **ルーブリックとしての機能**: 次元の違いはあるが、記述してみると既にルーブリックとして機能する形になっている

## 3. データモデル

### 3.1 ストレージキー設計

```
current_user                          // ログイン中ユーザー情報
students_list                         // 登録済み学生一覧（メンター向け）
survey:{userID}:{timestamp}           // 学生の自己評価アンケート
log:{userID}:{timestamp}              // YWT ログ
mentor_survey:{studentID}:{timestamp} // メンターによる学生評価
mjob:{corpId}                         // 求人分析結果キャッシュ（job-analyzer 連携時）
```

ポートフォリオはアンケートデータから動的に生成するため、専用ストレージキーなし。

### 3.2 ユーザー（current_user）

```js
{
  name: string,
  id:   string,
  role: 'student' | 'mentor',
}
```

### 3.3 学生リスト（students_list）

```js
[{ id: string, name: string, registeredAt: number }]
```

学生ロールでログインした際に自動登録。メンター画面の学生一覧に使用。

### 3.4 アンケート（survey）

```js
{
  userID:    string,
  timestamp: number,
  term:      string,   // 今期の目標
  axes: {
    1: number,   // Lv1〜4（calcAxesFromAnswers で survey_questions.json の axisWeights から算出）
    2: number,
    // ... 3〜9
  }
}
```

`axes` のキーは整数（`1`〜`9`）。値は `axisWeights` と回答値から加重平均で算出した 1〜4 の整数。

### 3.5 ログ（log）

```js
{
  userID:    string,
  timestamp: number,
  yatta:     string,         // Y: やったこと
  wakatta:   string,         // W: わかったこと
  tsugi:     string,         // T: 次にやること
  emotion:   1|2|3|4|5,     // 感情記録（1:😢 〜 5:😄）
}
```

### 3.6 メンター評価（mentor_survey）

```js
{
  mentorID:    string,
  mentorName:  string,
  studentID:   string,
  studentName: string,
  timestamp:   number,
  note:        string,   // 評価メモ（任意）
  axes: {
    1: number,   // Lv1〜4
    // ... 2〜9
  }
}
```

## 4. 機能一覧

### 4.1 ロール別画面構成

#### 学生（role: 'student'）

| 画面 | ナビ | 用途区分 | 機能概要 |
|---|---|---|---|
| ホーム | 「ホーム」 | — | レーダーチャート + 成長ポイント。クイックアクションを **📅 毎日やること / 🎯 節目にやること** に2分割表示 |
| アンケート入力 | 「評価」 | 🎯 節目 | `survey_questions.json` から設問を fetch。セクション別進捗バー。上部に「節目向け」使用文脈バナー表示 |
| 日次ログ（YWT） | 「日次」 | 📅 毎日 | YWT法 + 感情記録。上部に「毎日やること」使用文脈バナー表示 |
| ポートフォリオ | 「成果物」 | 🎯 節目 | アンケートから**自動生成**。9軸レベルバー・強み（Lv3+）・成長（Lv1-2）・レーダーチャート |
| 振り返り | 「振り返り」 | 🎯 節目 | サブタブ: **振り返り入力** / **問いへの回答**（メンターからの問い）/ **フィードバック**。上部に「節目向け」バナー |

#### メンター（role: 'mentor'）

| 画面 | ナビ | 機能概要 |
|---|---|---|
| 学生/評価 | 「学生/評価」 | 学生カード一覧 + 9軸評価入力フォーム。各評価軸名をタップで**ルーブリックポップアップ**表示 |
| 採点 | 「採点」 | 学生の振り返り提出一覧 → 採点。各評価軸名をタップでルーブリックポップアップ表示 |
| 問い | 「問い」 | 担当学生への問い投稿 + 全学生の問い一覧表示 |

### 4.2 アンケート設問方式（survey_questions.json）

各設問は複数の評価軸への寄与度（`axisWeights`）を持つ。回答値（1〜4）と `axisWeights` の加重平均で軸スコアを算出（`calcAxesFromAnswers`）。

```json
{
  "sections": [
    {
      "title": "セクションタイトル",
      "questions": [
        {
          "id": "q1",
          "text": "設問文",
          "options": ["選択肢1", "選択肢2", "選択肢3", "選択肢4"],
          "axisWeights": { "1": 0.8, "3": 0.2 }
        }
      ]
    }
  ]
}
```

`survey_questions.json` が `public/` に存在しない場合、アンケート画面にエラーメッセージを表示（フォールバックなし）。

### 4.3 チュートリアル機能（v1.5 実装）

初回ログイン時に自動表示。ヘッダーの「チュートリアル」ボタンで再表示可能。ロール別に内容が異なる。

| ロール | ステップ数 | 内容 |
|---|---|---|
| 学生 | 5ステップ | ホーム / アンケート / 振り返り / 問い の使い方を1画面ずつ説明 |
| メンター | 4ステップ | 学生/評価 / 採点 / 問い の使い方 |

- `localStorage` の `tutorial_seen` フラグで初回表示を制御。
- ドット型のステップインジケーター付き。テキストサイズは 15px・行間 2.0。

### 4.4 ルーブリックポップアップ（v1.5 実装）

メンターが採点・評価入力画面で各評価軸名をタップすると中央モーダルが開き、Lv.1〜4 の採点基準を表示。

```
RUBRIC_DATA = {
  1: { levels: [Lv1テキスト, Lv2, Lv3, Lv4] },  // 課題設定力
  2: { levels: null },   // 情報活用力（0626ルーブリックに記載なし → 「基準未確定」表示）
  3: { levels: [...] },
  ...
  8: { levels: null },   // 自律・内発的動機（同上）
  9: { levels: [...] },
}
```

- `RUBRIC_DATA[id].levels === null` の場合は「この評価軸は詳細な採点基準は未確定です」と表示。
- 背景クリックまたは「×」ボタンで閉じる。

### 4.5 レーダーチャートの表示ロジック

recharts `RadarChart` で9評価軸を正多角形表示。

| 表示場所 | 表示条件 | 内容 |
|---|---|---|
| **ホーム画面** | アンケートが1件以上あるとき | 自己評価（紫 `#CC44FF`）。メンター評価があれば水色 `#00E5FF` で重ね表示 |
| **ポートフォリオ** | アンケートが1件以上あるとき | 同上 |
| **メンター評価画面** | 学生選択後・自己評価が1件以上あるとき | 同上 |

### 4.6 ポートフォリオ自動生成ロジック

アンケート保存時に動的生成（専用ストレージなし）。

| カード要素 | 内容 |
|---|---|
| 今期の目標 | アンケートで入力した `term` |
| 9軸レベルバー | 全軸スコアを色付きバーで一覧 |
| 強みのエリア | Lv3以上の軸 → 軸名 + 根拠となる選択肢テキスト |
| 成長のエリア | Lv1〜2の軸 → 軸名 + 根拠となる選択肢テキスト |
| レーダーチャート | 自己評価 vs メンター評価 |

## 6. 設計上の判断

### 6.1 単一ファイル構成（POC）

`src/App.jsx` 1ファイルに全コードを集約。POC 段階のため保守性より全体把握のしやすさを優先。AI（Claude）に全文渡してデバッグ・拡張しやすい構成。本番化時はコンポーネント分割を推奨。

### 6.2 `localStorage` の採用

元バージョン（`be_ready_poc_app.jsx`）の `window.storage`（Claude Artifacts 専用 API）を `localStorage` に変更。`react-scripts` ベースのローカル開発・Vercel デプロイに対応するため。`storage` ヘルパー関数に集約されており、将来的な DB 移行が容易。

### 6.3 アンケートを JSON 外部定義方式に

設問・選択肢・軸への寄与度（`axisWeights`）を `public/survey_questions.json` で管理。アプリを再ビルドせずに設問を更新でき、0626ルーブリックの改訂に迅速に対応できる。

### 6.4 ポートフォリオをアンケートから自動生成

手動入力ではなくアンケート回答から自動生成することで、重複入力を排除。強みのエリア・成長のエリアの表示により、自己認識を促す設計。

### 6.5 ②⑧の扱い（v1.5 で本採点に復帰）

v1.4では `AXES` 定数の `ref: true` フラグで参考値表示していたが、v1.5で `ref: false` に戻し全軸本採点として扱う。ただし `RUBRIC_DATA` の ②⑧ は `levels: null` のままのため、ルーブリックポップアップでは「詳細基準は未確定」と表示される。0626ルーブリックに基準が記載された時点で `levels` 配列を追加するだけで対応可能。

### 6.6 認証を省略

POC 段階では氏名・ID の入力のみで「ログイン」。セキュリティ実装はスコープ外。本番化時は Firebase Authentication 等を推奨。
