# Be-Ready 評価 POC アプリ

北海学園大学 佐藤教授との共同プロジェクト「Be-Ready人材育成プログラム」における評価軸の実証実験（POC）用 Web アプリ。

**コンセプト**: アンケート（自己評価）+ ログ（YWT）+ 成果物（ポートフォリオ）の3要素で、Be-Ready人材の評価基準を体験・検証する。

## 1. プロダクト概要

**作成日**: 2026年5月26日
**プロダクト**: Be-Ready 評価 POC アプリ
**バージョン**: v1.0（POC）
**対象ユーザー**: 学生 / メンター

### 1.1 背景

座学教育との差別化を目的に、企業が「即戦力かどうか」を判断できる評価軸（9項目）を定義。その評価軸を実際に運用・検証するための POC ツールとして開発。

### 1.2 評価の3要素

| 要素 | 内容 | 評価への寄与 |
|---|---|---|
| アンケート | 9評価軸 × 4レベルのルーブリック自己判定 | 定量的スコア・成長可視化 |
| ログ | YWT法（やったこと / わかったこと / 次にやること）による振り返り | 思考プロセスの記録 |
| 成果物 | ポートフォリオへの作品・レポート蓄積 | 学習の軌跡と文脈の把握 |

### 1.3 評価軸9項目

| 番号 | 軸名 | 分類 |
|---|---|---|
| ① | 課題設定力 | A 思考・判断系 |
| ② | 情報活用力 | A 思考・判断系 |
| ③ | 不確実性への耐性 | A 思考・判断系 |
| ④ | 提案・発信力 | B 行動・実行系 |
| ⑤ | 実行・改善力 | B 行動・実行系 |
| ⑥ | オーナーシップ | B 行動・実行系 |
| ⑦ | 協働・調整力 | C 関係・姿勢系 |
| ⑧ | 自律・内発的動機 | C 関係・姿勢系 |
| ⑨ | 行動変容力 | C 関係・姿勢系 |

### 1.4 評価レベル（4分類）

| Lv | 名称 | 理解の種類 | 特徴 |
|---|---|---|---|
| Lv.1 | 受動性 | 「知る」理解 | 外発的動機で動く |
| Lv.2 | 能動性 | 「分かる」理解 | 自ら動くが一時的 |
| Lv.3 | 自律性 | 「納得」の理解 | 自分事化して継続 |
| Lv.4 | 創造性 | 「本質」の理解 | 新しさを生む |

## 2. 技術スタック・ファイル構成

| カテゴリ | 採用技術 | 補足 |

## 3. データモデル

### 3.1 ストレージキー設計

```
current_user                      // ログイン中ユーザー情報
survey:{userID}:{timestamp}       // アンケート回答（9軸 × レベル判定）
log:{userID}:{timestamp}          // YWT ログ
portfolio:{userID}:{timestamp}    // 成果物（ポートフォリオ）
```

データはすべて個人スコープ（`shared: false`）で保持。

### 3.2 ユーザー

```js
{
  name: string,         // 氏名
  id: string,           // ユーザーID
  role: 'student' | 'mentor',
}
```

### 3.3 アンケート（Survey）

```js
{
  userID: string,
  timestamp: number,
  term: string,         // 今期の目標
  axes: {
    [axisKey: string]: {
      level: 1 | 2 | 3 | 4,   // ルーブリック判定レベル
      comment: string,          // 補足コメント
    }
  }
}
```

`axisKey` は `axis1` 〜 `axis9`（①課題設定力〜⑨行動変容力）に対応。

### 3.4 ログ（YWT Log）

```js
{
  userID: string,
  timestamp: number,
  yatta: string,        // やったこと
  wakatta: string,      // わかったこと
  tsugi: string,        // 次にやること
  emotion: 1 | 2 | 3 | 4 | 5,   // 感情記録（1:とても辛い 〜 5:とても良い）
}
```

### 3.5 ポートフォリオ（Portfolio）

```js
{
  userID: string,
  timestamp: number,
  title: string,           // 成果物タイトル
  type: string,            // 種類（レポート / 提案資料 / コード / その他）
  url: string,             // 参照リンク
  relatedAxes: string[],   // 関連する評価軸（axisKey の配列）
  reflection: string,      // 振り返りメモ
}
```

## 4. 機能一覧

### 4.1 画面構成（4画面）

| 画面 | 役割 |
|---|---|
| ログイン画面 | 氏名・ID・区分（学生 / メンター）の入力 |
| ホーム（ダッシュボード） | 評価基準の3要素説明・統計サマリ・クイックアクション |
| アンケート入力 | 今期目標の設定 + 9軸 × 4レベルのルーブリック自己判定 |
| ログ入力 | YWT 法による振り返り記録 + 感情記録 + 過去ログ一覧 |
| ポートフォリオ | 成果物の追加・カード一覧表示 |

### 4.2 ホーム画面

- 評価基準の3要素（アンケート / ログ / 成果物）を可視化して提示
- 統計サマリ：平均レベル、アンケート件数、ログ件数、成果物件数
- クイックアクション：各入力画面への導線

### 4.3 アンケート入力画面

- **今期の目標**設定欄（目標設定起点アプローチ）
- 評価軸9項目 × 4レベルのルーブリック判定（ラジオ選択）
- 各軸への補足コメント入力
- リアルタイム進捗・平均レベル表示
- 回答保存（`window.storage` へ永続化）

### 4.4 ログ入力画面

- YWT 法（やったこと / わかったこと / 次にやること）のテンプレート入力
- 感情記録（5段階：😢 〜 😄）
- 過去ログの一覧表示（新しい順）

### 4.5 ポートフォリオ画面

- 成果物の追加（タイトル / 種類 / リンク / 関連評価軸 / 振り返りメモ）
- カード形式の一覧表示（新しい順）

## 5. デプロイ手順

### 前提

- Google アカウント（GitHub・Vercel の作成に使用）
- Node.js 18 以上（ローカルビルドする場合）
- 所要時間：20〜30 分

### Step 1: GitHub アカウント作成（未作成の場合）

1. https://github.com/ にアクセス
2. 「Sign up」→ メール・パスワード設定
3. メール認証を完了

### Step 2: リポジトリ作成

1. ログイン後、右上「+」→「New repository」
2. Repository name: `be-ready-poc`（任意）
3. Public / Private どちらでも可（個人利用なら Private 推奨）
4. 「Create repository」

### Step 3: React プロジェクトのセットアップ

```bash
npx create-react-app be-ready-poc
cd be-ready-poc
npm install lucide-react
```

`src/` 配下を以下の構成に置き換える：

```
src/
├── App.jsx          ← be_ready_poc_app.jsx の内容をコピー
├── index.js
└── index.css
```

> **注意**: `window.storage` API は Claude Artifacts 専用。本番移行時は `localStorage` または Firebase などに差し替えが必要（[7章](#7-今後の課題拡張予定) 参照）。

### Step 4: GitHub にプッシュ

```bash
git init
git add .
git commit -m "Initial commit: Be-Ready POC app"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/be-ready-poc.git
git push -u origin main
```

### Step 5: Vercel でデプロイ

1. https://vercel.com/ にアクセス
2. 「Sign Up」→「Continue with GitHub」
3. ダッシュボードで「Add New...」→「Project」
4. `be-ready-poc` リポジトリを選択して「Import」
5. 設定はデフォルトのまま「Deploy」
6. 2〜3 分でビルド完了 → URL が発行される

### アプリの更新

GitHub にファイルをプッシュすると Vercel が自動で再デプロイ（30秒〜2分）。

## 6. 設計上の判断

### 6.1 単一ファイル構成（POC）

`be_ready_poc_app.jsx` 1 ファイルに全コードを集約している理由：

- POC 段階のため、コンポーネント分割より全体把握のしやすさを優先
- Claude Artifacts 上で即時動作確認できる
- AI（Claude）に全文渡してデバッグ・拡張しやすい

トレードオフ: 機能追加に伴い可読性・保守性が低下する。本番化時はコンポーネント分割を推奨。

### 6.2 `window.storage` API の採用

Claude Artifacts 専用のストレージ API。`localStorage` の代替として使用しており、個人スコープ（`shared: false`）でデータを永続化している。

**本番化時の対応**: `window.storage` を `localStorage` またはクラウド DB（Firebase / Supabase）に差し替える。ストレージ呼び出しは各コンポーネントに分散しているため、移行時はヘルパー関数に集約してから差し替えることを推奨。

### 6.3 認証を省略した理由

POC 段階では氏名・ID の入力のみで「ログイン」とし、パスワード認証・セッション管理は実装していない。

理由: 評価フローの体験検証が目的であり、セキュリティ実装はスコープ外。本番化時は Firebase Authentication 等の導入を推奨。

### 6.4 目標設定起点アプローチ

アンケート画面の冒頭に「今期の目標」設定欄を設けている。これはアンケート（評価者起点）から「自分で目標を立て、達成度を自ら振り返る」（学習者起点）への転換を体現する設計で、自律性（Lv.3）の育成と評価を兼ねている。

### 6.5 TypeScript 不採用

POC の軽量さと開発速度を優先して TypeScript は採用していない。本番化・チーム開発時には TypeScript 化を推奨。

## 7. 今後の課題・拡張予定

### 7.1 POC で未対応の機能

| 項目 | 優先度 | 備考 |
|---|---|---|
| メンター用ダッシュボード | 高 | 担当学生の一覧・Lv 推移・ログ閲覧 |
| `window.storage` → `localStorage` 差し替え | 高 | 本番移行の前提 |
| 認証（パスワード / OAuth） | 高 | Firebase Auth 等の導入 |
| 自己評価 × 他者評価のギャップ可視化 | 中 | ジョハリの窓的レーダーチャート |
| AI 分析機能（STAGE 2） | 中 | 目標と振り返りの整合分析、Lv 候補推定 |
| データエクスポート | 中 | CSV / JSON 出力 |
| 心理テスト型 UI（ゲーム性） | 低 | アンケートのエンゲージメント向上 |
| ナラティブ・インタビュー記録 | 低 | MSC / リフレクティブジャーナル対応 |

### 7.2 AI 評価ロードマップ（スライド39に対応）

| ステージ | 内容 | 現在地 |
|---|---|---|
| STAGE 1 | AI と人で評価（並走） | ← **現在** |
| STAGE 2 | AI がデータ集約・分析 → 人が最終判定 | 次フェーズ |
| STAGE 3 | AI が評価を自律判定 → 人は監督 | 将来構想 |

### 7.3 システム設計の2軸

今後の検討は以下の2軸で進める：

- **評価設計**（枠組み）: 評価軸・ルーブリックの精緻化、ナラティブ評価の組み込み
- **システム実装**（仕組み）: データ基盤設計、AI 分析機能、メンター向け UI
