# Be-Ready 評価 POC アプリ

北海学園大学 佐藤教授との共同プロジェクト「Be-Ready人材育成プログラム」における評価軸の実証実験（POC）用 Web アプリ。

**コンセプト**: アンケート（自己評価）+ ログ（YWT）+ ポートフォリオの3要素で、Be-Ready人材の評価基準を体験・検証する。

## 1. プロダクト概要

**作成日**: 2026年5月26日  
**最終更新**: 2026年6月8日（ホーム画面レーダーチャート追加・UI カラー改善）  
**バージョン**: v1.2（POC）  
**対象ユーザー**: 学生 / メンター

### 1.1 背景

座学教育との差別化を目的に、企業が「即戦力かどうか」を判断できる評価軸（9項目）を定義。その評価軸を実際に運用・検証するための POC ツールとして開発。

### 1.2 評価の3要素

| 要素 | 内容 | 評価への寄与 |
|---|---|---|
| アンケート | 9評価軸 × 4レベルの記述式4択による自己判定 | 定量的スコア・成長可視化 |
| ログ | YWT法（やったこと / わかったこと / 次にやること）による振り返り | 思考プロセスの記録 |
| ポートフォリオ | アンケート回答から自動生成・強み／成長エリアの可視化 | 学習の軌跡と文脈の把握 |

### 1.3 評価軸9項目

| 番号 | 軸名 | 分類 |
|---|---|---|
| ① | 課題設定力 | A 思考・判断系 |
| ② | 情報活用力 | A 思考・判断系 |
| ③ | 不確実性への耐性 | A 思考・判断系 |
| ④ | 提案・発信力 | B 行動・実行系 |
| ⑤ | 実行・改善力 | B 行動・実行系 |
| ⑥ | オーナーシップ | B 行動・実行系 |
| ⑦ | 協働・調整力 | C 関係・姿勢系 |
| ⑧ | 自律・内発的動機 | C 関係・姿勢系 |
| ⑨ | 行動変容力 | C 関係・姿勢系 |

### 1.4 評価レベル（4分類）

| Lv | 名称 | 理解の種類 | 特徴 |
|---|---|---|---|
| Lv.1 | 受動性 | 「知る」理解 | 外発的動機で動く |
| Lv.2 | 能動性 | 「分かる」理解 | 自ら動くが一時的 |
| Lv.3 | 自律性 | 「納得」の理解 | 自分事化して継続 |
| Lv.4 | 創造性 | 「本質」の理解 | 新しさを生む |

## 3. データモデル

### 3.1 ストレージキー設計

```
current_user                         // ログイン中ユーザー情報
students_list                        // 登録済み学生一覧（メンター向け）
survey:{userID}:{timestamp}          // 学生の自己評価アンケート
log:{userID}:{timestamp}             // YWT ログ
mentor_survey:{studentID}:{timestamp} // メンターによる学生評価
```

ポートフォリオはアンケートデータから動的に生成するため、専用キーは存在しない。

### 3.2 ユーザー（current_user）

```js
{
  name: string,
  id: string,
  role: 'student' | 'mentor',
}
```

### 3.3 学生リスト（students_list）

```js
[
  { id: string, name: string, registeredAt: number }
]
```

学生ロールでログインした際に自動登録される。メンター画面の学生一覧に使用。

### 3.4 アンケート（survey）

```js
{
  userID: string,
  timestamp: number,
  term: string,       // 今期の目標
  axes: {
    axis1: { level: 1|2|3|4, comment: string },
    axis2: { level: 1|2|3|4, comment: string },
    // ... axis3 〜 axis9
  }
}
```

`level` は記述式4択の選択肢インデックス（1〜4）に対応し、そのまま評価レベルとなる。

### 3.5 ログ（log）

```js
{
  userID: string,
  timestamp: number,
  yatta: string,    // Y: やったこと
  wakatta: string,  // W: わかったこと
  tsugi: string,    // T: 次にやること
  emotion: 1|2|3|4|5,  // 感情記録（1:😢 〜 5:😄）
}
```

### 3.6 メンター評価（mentor_survey）

```js
{
  mentorID: string,
  mentorName: string,
  studentID: string,
  studentName: string,
  timestamp: number,
  note: string,     // 評価メモ（任意）
  axes: {
    axis1: { level: 1|2|3|4, comment: string },
    // ... axis3 〜 axis9
  }
}
```

## 4. 機能一覧

### 4.1 ロール別画面構成

#### 学生（role: 'student'）

| 画面 | キー | 機能概要 |
|---|---|---|
| ホーム | `home` | 統計カード（アンケート・ログ件数）+ **Be-Ready 評価レーダーチャート**（アンケートがあれば表示）+ クイックアクション |
| アンケート入力 | `survey` | 今期の目標 + 9軸 × 記述式4択。カテゴリ別（A/B/C）表示。過去回答履歴 |
| ログ（YWT） | `log` | やったこと・わかったこと・次にやること + 感情記録。過去ログ一覧・削除 |
| ポートフォリオ | `portfolio` | アンケートから自動生成。目標・9軸レベルバー・強み/成長エリア + **レーダーチャート**（自己評価 vs メンター評価）を表示 |

#### メンター（role: 'mentor'）

| 画面 | キー | 機能概要 |
|---|---|---|
| 学生一覧ホーム | `home` | 登録済み学生カード。平均Lv・アンケート件数・ログ件数を表示 |
| 学生詳細 | `student_detail` | タブ切り替え（アンケート / ログ / 成果物）で学生データを閲覧（読み取り専用） |
| メンター評価入力 | `mentor_survey` | 学生を選択して9軸評価を入力。評価履歴表示。`mentor_survey` キーで別保存 |

### 4.2 アンケート入力の設問設計

各評価軸に対して質問文と4つの記述式選択肢を用意。学生はラジオボタン形式で選択し、選択肢のインデックス（1〜4）がそのまま評価レベルとなる。

| 軸 | 質問文（要約） |
|---|---|
| ①課題設定力 | 取り組む課題や「何を解くか」の設定について |
| ②情報活用力 | 情報の収集・整理・活用について |
| ③不確実性への耐性 | 答えが明確でない状況・曖昧な場面での行動について |
| ④提案・発信力 | 自分の意見や考えを伝えることについて |
| ⑤実行・改善力 | 取り組みをやり切り、改善することについて |
| ⑥オーナーシップ | 活動への主体性・責任感について |
| ⑦協働・調整力 | チームやメンバーとの協力・調整について |
| ⑧自律・内発的動機 | 活動に取り組む動機・理由について |
| ⑨行動変容力 | フィードバックを受けての変化・成長について |

### 4.3 レーダーチャートの表示ロジック

9評価軸のスコアを正多角形（レーダー）で可視化する。recharts ライブラリを使用。

| 表示場所 | 表示条件 | 内容 |
|---|---|---|
| **ホーム画面** | アンケートが1件以上あるとき | 最新の自己評価（紫 `#CC44FF`）。メンター評価がある場合は水色 `#00E5FF` で重ね表示 |
| **ポートフォリオ画面** | アンケートが1件以上あるとき | 同上 |
| **メンター評価画面** | 学生選択後・自己評価が1件以上あるとき | 同上 |

軸ラベル（課題・情報・不確実・提案・実行・所有・協働・動機・変容）はコンパクト表記で表示。

### 4.4 ポートフォリオの自動生成ロジック

アンケートを保存すると、その回答をもとにポートフォリオカードが自動生成される。

カードに含まれる情報：

| 項目 | 内容 |
|---|---|
| 今期の目標 | アンケートで入力した目標文 |
| 9軸レベルバー | 全軸のレベルを色付きバーで一覧表示 |
| 強みのエリア | Lv3以上の軸：軸名 + 選択した回答文 |
| 成長のエリア | Lv1〜2の軸：軸名 + 選択した回答文 |
| 平均レベル | 全回答軸の平均値 |

### 4.4 メンター機能

- 学生がログインすると `students_list` に自動登録される
- メンターの学生詳細画面では、自己評価とメンター評価を混在表示（「メンター」バッジで区別）
- メンター評価は `mentor_survey` キーで保存され、学生の自己評価とは独立して管理される

## 6. 設計上の判断

### 6.1 単一ファイル構成（POC）

`src/App.jsx` 1ファイルに全コードを集約している理由：

- POC 段階のため、コンポーネント分割より全体把握のしやすさを優先
- AI（Claude）に全文渡してデバッグ・拡張しやすい

トレードオフ: 機能追加に伴い可読性・保守性が低下する。本番化時はコンポーネント分割を推奨。

### 6.2 `localStorage` の採用

上司作成版（`be_ready_poc_app.jsx`）は `window.storage`（Claude Artifacts 専用の非同期 API）を使用しているが、`src/App.jsx` では `localStorage` を採用している。

理由：
- `window.storage` は Claude Artifacts 環境外では動作しない
- `react-scripts` ベースのローカル開発環境・Vercel デプロイに対応するため
- 同期 API のほうがシンプルで POC 段階に適している

**本番化時**: `localStorage` → Firebase / Supabase 等のクラウド DB への移行を推奨。ストレージ呼び出しは `storage` ヘルパー関数（[App.jsx:126](src/App.jsx#L126)）に集約されているため、差し替えが容易。

### 6.3 インラインスタイルの採用

上司作成版は Tailwind CSS を使用しているが、`src/App.jsx` ではインラインスタイルを採用している。

理由：
- ビルド環境（Node.js v11）の制約で Tailwind の PostCSS 設定が困難
- インラインスタイルで CSS 変数（`var(--bg)` 等）を使い、ダークモード対応の余地を残している

### 6.4 ポートフォリオの自動生成

上司作成版は手動入力方式だが、`src/App.jsx` ではアンケート回答から自動生成する方式を採用。

理由：
- 「アンケートの内容をもとに作成」という要件に対応
- 重複入力を排除し、アンケートを書けば自動的に学習記録が蓄積される体験を実現
- 強みのエリア・成長のエリアの表示により、自己認識を促す設計

### 6.5 記述式4択アンケート

上司作成版は Lv1〜4 のラベルと説明文を直接選ぶ方式だが、`src/App.jsx` では各軸に対応した質問文＋記述式4択を採用。

理由：
- 学生が「Lv3とはどういう状態か」を知らなくても回答できる
- 自分の行動・状態を説明文と照合することで、より正確な自己評価が期待できる
- 回答プロセス自体がリフレクションになる

### 6.6 認証を省略した理由

POC 段階では氏名・ID の入力のみで「ログイン」とし、パスワード認証・セッション管理は実装していない。

理由: 評価フローの体験検証が目的であり、セキュリティ実装はスコープ外。本番化時は Firebase Authentication 等の導入を推奨。

## 7. 今後の課題・拡張予定

### 7.1 2バージョンの統合方針

現在、以下の2バージョンが存在する。

| ファイル | 特徴 | 状態 |
|---|---|---|
| `src/App.jsx` | メンター機能・記述式アンケート・自動生成ポートフォリオ | 稼働中 |
| `be_ready_poc_app.jsx` | PC サイドバー UI・Tailwind・window.storage | 参照用 |

**統合方針（検討中）**:

| 論点 | 候補A | 候補B |
|---|---|---|
| レイアウト | モバイル優先（現状維持） | PC サイドバーに統一 or レスポンシブ化 |
| スタイリング | インラインスタイル維持 | Tailwind 導入（Node.js 更新が前提） |
| ポートフォリオ | 自動生成のみ（現状） | 自動生成 + 手動追加の併用 |
| アンケート UI | 記述式4択（現状） | Lv ボタン形式との選択 or 切り替え |

### 7.2 未対応機能（優先度順）

| 項目 | 優先度 | 備考 |
|---|---|---|
| 認証（パスワード / OAuth） | 高 | Firebase Auth 等の導入 |
| `localStorage` → クラウド DB 移行 | 高 | Firebase / Supabase を想定 |
| 自己評価 × メンター評価のギャップ可視化（強調） | 中 | 現在レーダーで重ね表示済み。差分ハイライト・コメント表示を追加予定 |
| AI 分析機能（STAGE 2） | 中 | 目標と振り返りの整合分析、Lv 候補推定 |
| データエクスポート | 中 | CSV / JSON 出力 |
| PC 向けサイドバーレイアウト | 低〜中 | 上司作成版の UI を取り込む |
| 心理テスト型 UI（ゲーム性） | 低 | アンケートのエンゲージメント向上 |
| ナラティブ・インタビュー記録 | 低 | MSC / リフレクティブジャーナル対応 |

### 7.3 AI 評価ロードマップ（スライド39に対応）

| ステージ | 内容 | 現在地 |
|---|---|---|
| STAGE 1 | AI と人で評価（並走） | ← **現在** |
| STAGE 2 | AI がデータ集約・分析 → 人が最終判定 | 次フェーズ |
| STAGE 3 | AI が評価を自律判定 → 人は監督 | 将来構想 |
