# Be-Ready 評価アプリ — 開発計画・費用概算

> 作成日：2026年7月2日  
> 担当：谷川 賢治（Accenture）  
> 共同：北海学園大学 佐藤教授

---

## 1. プロジェクト概要

北海学園大学PBLにおいて、学生の「Be-Ready人材」としての成長を可視化・促進するWebアプリのPOCを開発中。  
9つの評価軸×4段階レベルのルーブリックに基づき、学生の自己評価・メンター評価・AI分析を統合する。

### 現状

- **フロントエンド**：React 18（App.jsx 約1,750行）— 実装済み
- **データ保存**：ブラウザのlocalStorage（端末ローカル）— **要改善**
- **AI機能**：メンター採点補助（設計段階）— **未実装**

### 課題

| 課題 | 内容 |
|------|------|
| データ消失リスク | ブラウザキャッシュ削除・端末変更でデータが消える |
| データ共有不可 | 学生とメンターが同じデータをリアルタイムで参照できない |
| スケーラビリティ | 複数チーム・複数期での継続利用が困難 |

## 2. 今後の実装ステップ

---

### Phase 2：バックエンド構築（2〜3日）

#### 技術構成（AWS Amplify）

```
[ブラウザ / スマホ]
       │
       │ HTTPS
       ▼
[AWS Amplify Hosting]  ← Reactアプリのホスティング
       │
       │ API呼び出し
       ▼
[Amazon API Gateway]   ← APIエンドポイント
       │
       ▼
[AWS Lambda]           ← サーバーレス処理関数（Node.js）
       │
       ▼
[Amazon DynamoDB]      ← データベース（NoSQL）
```

#### 保存するデータ一覧

| テーブル | 内容 |
|----------|------|
| `users` | ユーザー（ID・名前・役割・チームID） |
| `surveys` | 学生アンケート回答・評価軸スコア |
| `mentor_surveys` | メンター採点結果 |
| `questions` | メンターから学生への問い・学生の回答 |
| `next_actions` | 学生のネクストアクション |
| `logs` | 活動ログ |

---

### Phase 3：Gemini AI 連携（2〜3日）

#### 利用シーン

**① メンター向け：採点補助**
- 学生の振り返りテキストをGemini APIに送信
- 9評価軸ごとのスコア提案をAIが返す
- メンターが確認・修正して保存

**② 学生向け：自己分析支援**
- 学生が自分のアンケート回答・評価履歴を送信
- 「あなたの強み・成長ポイント」をAIが文章で返す
- レーダーチャートと合わせて表示

#### 実装方法

```
学生/メンター
    │ 振り返りテキスト送信
    ▼
[Reactアプリ]
    │ API呼び出し
    ▼
[Lambda関数]  ← Gemini APIキーをサーバー側で管理（セキュア）
    │
    ▼
[Google Gemini 1.5 Flash API]
    │ 分析結果（JSON）
    ▼
[アプリに表示]
```

## 3. 費用概算

### AWS 利用料金（月次）

| サービス | 無料枠 | POC想定使用量 | 概算費用 |
|----------|--------|---------------|----------|
| DynamoDB | 25GB・読み書き無制限（**永続無料**） | 数MB・数千回/月 | **$0** |
| Lambda | 100万リクエスト/月（**永続無料**） | 数千回/月 | **$0** |
| API Gateway | 100万リクエスト/月（12ヶ月無料） | 数千回/月 | **$0** |
| Amplify Hosting | 15GB転送・1,000分ビルド（12ヶ月） | 数MB | **$0** |
| **合計（POC期間）** | | | **$0〜数ドル/月** |

> ※ AWSアカウント作成にクレジットカードの登録が必要。ただし、無料枠内の利用であれば請求は発生しない。

### Gemini API 利用料金

| モデル | 無料枠 | 想定使用量 | 概算費用 |
|--------|--------|------------|----------|
| Gemini 1.5 Flash | 60リクエスト/分・1,500回/日（**無料**） | 数十回/日 | **$0** |

> ※ 無料枠を超えた場合：$0.075/百万トークン（入力）。POC規模では超過しない見込み。

### 費用サマリー

| 項目 | POC期間（〜12ヶ月）| 12ヶ月以降 |
|------|-------------------|------------|
| AWS インフラ | **$0〜数ドル/月** | $10〜50/月程度 |
| Gemini API | **$0** | $0〜数ドル/月 |

## 5. リスクと対策

| リスク | 対策 |
|--------|------|
| AWS無料枠超過 | 予算アラート設定（$1超えたらメール通知） |
| Gemini API制限 | 無料枠のリクエスト数をモニタリング |

| Node.js更新でのビルドエラー | テスト環境で確認後に本番適用 |

## 6. 上司相談用サマリー（1枚）

### 何をするか
北海学園大学PBLの学生評価アプリを、**ローカル保存から本番運用可能なクラウドバックエンドへ移行**する。

### なぜ今か
現在はブラウザのローカルストレージに保存しており、端末を変えるとデータが消える。複数チームへの展開・継続的なデータ蓄積のためにサーバーが必要。

### いくらかかるか
- **POC期間（〜12ヶ月）：実質$0〜数ドル/月（≒0〜数百円/月）**
- AWS無料枠・Gemini無料枠の範囲内で収まる見込み
- 本格展開時（数百人規模）：月額$10〜50程度

### 何が必要か
- AWSアカウントの作成（クレジットカード登録必要、請求は発生しない見込み）
- 開発工数：約1〜2週間（社内工数のみ）

### 判断ポイント
AWSアカウントをAccentureの個人または組織アカウントで作成してよいか。
